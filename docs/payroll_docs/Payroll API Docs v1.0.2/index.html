    Payroll Solution API Documentation

    Documentation of APIs for the Simplified Payroll Solution.

    Version: v 1.0.1

    Base URL (For Development)

    https://kratest.pesaflow.com:30000/api/

    Caveat: employer_tax_payer_id and organization_id are used interchangeably, depending on whether the authenticated user is an organization or individual. This will usually be an individual.

    Setup: Authentication

    All API requests require authentication. Include the authentication token in the request header:

    Authorization: Bearer {your_token}

    The Postman Token Generation API Collection is included Token Generation API to get the authentication token.

    Token Validity: 15 minutes.

    Section 1: Employee Management

    1.0 Validate employee details

    Validate employee details before adding. Endpoint: GET payroll/employee/personal-details/validate

    Query params:

    {
  "pin": "A01*******Z",
  "gui": "107*****6",
  "type": "citizen",
  "first_name": "George",
  "employer_tax_payer_id": "21128", 
  "tax_payer_pin": "A0******09L"
}

    Success response:

    {
  "dob":"1968-01-01",
  "gender":"M",
  "gui":"107*****6",
  "name":"GEORGE KIRIMI MURITHI",
  "pin":"A0*******43Z",
  "type":"citizen"
}

    Error responses:

    • Invalid ID - ID does not match the PIN.
    • Name mismatch - Name submitted does not match the records.
    • Employee exists - Employee exists.

    1.1 Add Single Employee

    Endpoint: POST /payroll/employee

    Create a new employee record in the payroll system.

    Request Body: - As elaborated in Employee Payload Docs

    Response: Status Code: 201 (Created)

    {"message":"Employee added"}

    1.2 Add Bulk Employees

    Endpoint: POST /payroll/employee/upload-employees As defined in Employee Upload.md

    1.3 Edit Employee

    Get a single employee: GET /payroll/employee/:uuid

    Edit employee: POST /payroll/employee/:uuid/update

    Update an existing employee's information.

    Path Parameters: - uuid - Employee unique identifier

    Request Body: Similar to employee payload in Employee Payload Docs.md

    Response: Response string "employee updated".

    1.4 Deactivate Employee

    Endpoint: POST /payroll/employee/:uuid/toggle-status

    Toggle employee status (active/inactive).

    Path Parameters: - uuid - Employee unique identifier

    Request Body:

    {"status":false,"deactivation_reason":"Go on break"}

    Response:

    Response string: "employee updated"

    1.5 Activate Employee

    Endpoint: POST /payroll/employee/:uuid/toggle-status

    Toggle employee status (active/inactive).

    Path Parameters: - uuid - Employee unique identifier

    Request Body:

    {"status":true,"deactivation_reason":""}

    Response:

    Response string: "employee updated"

    1.6 Terminate Employee

    Endpoint: POST /payroll/employee/:uuid/remove

    Toggle employee status (active/inactive).

    Path Parameters: - uuid - Employee unique identifier

    Request Body:

    {"termination_reason":"End of contract","termination_date":"2025-12-04T04:32:00.000Z"}

    Response:

    Response string: "employee removed"

    1.7 Reinstate Employee

    Endpoint: POST /payroll/employee/:uuid/reinstate

    Reinstate a previously terminated employee.

    Path Parameters: - uuid - Employee unique identifier

    Request Body:

    {"new_employment_no":"SAMPLEEMP220"} // optional

    Response:

    Response string: "employee reinstated"

    Additional Employee Endpoints

    List All Employees

    Endpoint: GET /payroll/employee

    Query Parameters: - page - Page number (default: 1) - page_size - Results per page (default: 20) - status - Filter by status (active/inactive) - multi_filter - Search by name or PIN - contract_type - Array of contract types. ['permanent', 'fixed-term', consultant, ...]

    Get Single Employee

    Endpoint: GET /payroll/employee/:uuid

    Path Parameters: - uuid - Employee unique identifier

    Section 2: Process Payroll

    2.1 Process Regular Payroll

    Endpoint: POST /payroll

    Initiate a new payroll processing cycle for employees or consultants, based on payroll_type and contract_type fields.

    Request Body:

    {
  "period":"2025-09-30",
  "employer_tax_payer_id":"21128", // or organization_id of authenticated user
  "contract_type":["permanent","contract","internship","fixed-term","casual"], // or ['consultant']
  "payroll_type":"regular" // or 'consultants'
}

    Response:

    {"ref_no":"PR-HALLHV"}

    2.1.1 Upload Payroll File

    As outlined in Payroll Upload file

    2.1.2 Fetch payroll

    Endpoint: GET /payroll/:ref_no? Fetch the created payroll.

    Path Parameters: - ref_no - Payroll reference number

    Request Body:

    {
  "payroll_ref_no" : "PR-HALLHV",
  "page": "1",
  "page_size": "10"
}

    Response: Documented in Payroll Payload.md

    2.4 Update Payroll

    Endpoint: POST /payroll/:ref_no/update

    Update payroll. Mainly used to submit payroll to a 'Pending Filing' state.

    Path Parameters: - ref_no - Payroll reference number

    Request body:

    {"status":"pending_filing"}

    Additional Payroll Endpoints

    List Payrolls

    Endpoint: GET /payroll

    Query Parameters: - page, page_size, status, period, payroll_type

    Get Single Payroll

    Endpoint: GET /payroll/:ref_no

    Payroll Statistics

    Endpoint: GET /payroll/stats

    Get Available Periods

    Endpoint: GET /payroll/get-periods

    Query params: - employer_tax_payer_id - ID of authenticated user, if individual. - organization_id - ID of authenticated user, if organization.

    Get System Constants

    Endpoint: GET /payroll/get-constants

    Returns defined payroll constants.

    Section 3: Reports

    3.1 Download Payslip

    Endpoint: GET /payroll/employee/download-payslip/:ref_no

    Download individual employee payslip or voucher for consultants.

    Path Parameters: - ref_no - Payroll item reference number

    Response: PDF file

    3.2 Send Payslips

    Endpoint: POST /payroll/:ref_no/send-payslips

    Send payslips to all employees via email.

    Path Parameters: - ref_no - Payroll reference number

    3.3 Send Single Payslip

    Endpoint: POST /payroll/:ref_no/send-one-payslip

    Send payslip to a specific employee.

    Request Body:

    {
  "employee_uuid": "string",
  "email": "employee@example.com"
}

    Section 4: Filing

    4.1 Filing PAYE

    Endpoint: POST /payroll/:ref_no/file-return

    File PAYE return to KRA iTax system.

    Path Parameters: - ref_no - Payroll reference number

    Request Body: None.

    Success Response:

    {
  "message": "Return filed successfully with Ack. No."
}

    Error Response:

    {
  "errors": [
    "Return cannot be filed for this period",
    "Taxable amount for taxpayer with pin A000000000001 cannot be less than 20000",
    ...
  ]
}

    4.2 Filing WHT

    In development.

    4.3 Update PAYE Token

    Endpoint: POST /payroll/update-paye-token

    Update iTax authentication token.

    Request Body:

    {
  "token": "new_itax_token",
  "expires_at": "2024-12-31T23:59:59Z"
}

    Payment Generation

    Generate Payment

    Endpoint: POST /payroll/make-payment

    Generate payment for AHL or NITA. To make PAYE payments, access the liabilities endpoint.

    Request Body:

    {"ref_no":"PR-BEINRE","type":"AHL"}

    Sample body:

    "inserted_at": "2025-12-02T05:18:08",
    "payment": {
        "tax_return": null,
        "amount_paid": "0",
        "items": [
            {
                "account_id": 194207,
                "amount_expected": null,
                "desc": "3306",
                "id": "908dd73f-17dd-46b6-a034-a7cf0cd755a1",
                "item_ref": "ITM-MTSL",
                "price": "37500.0",
                "quantity": "1.0",
                "require_settlement": true,
                "settlements": {
                    "account_number": "194207",
                    "desc": "Affordable Housing Levy",
                    "id": "2dd1d26a-6fa1-4b47-a9b0-4f426cb5e7f5",
                    "value": "37500.0"
                }
            }
        ],
        "status": "pending",
        "period": "01/12/2025 - 31/12/2025",
        "transacted_by": null,
        "metadata": {},
        ...,
        "payment_date": "02/12/2025",
        "ref_no": "2020250002604798", // use the ref_no to view and pay for the payment
