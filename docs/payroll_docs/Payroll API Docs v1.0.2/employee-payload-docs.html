    Employee Payload Documentation

    Overview

    This document explains the payload structure for adding an employee to the payroll system.

    The payload json is found in Employee Payload JSON

    Core Employee Information

    Field Type Required Description
    name string Yes Full name of the employee
    pin string Yes KRA PIN number
    nssf_no string No NSSF registration number
    msisdn string No Employee phone number
    type string Yes Employee type. Options: "citizen", "resident", "non-resident"
    primary_employee boolean Yes Whether this is a primary employee record
    contract_type string Yes Options: "permanent", "fixed-term", "contract", "casual", "consultant"
    date_of_employment string (ISO 8601) Yes Employee start date
    date_of_completion string (ISO 8601) No Contract end date (required for fixed-term contracts)

    Income Structure

    Salary

    Field Type Required Description
    salary number Yes Basic salary amount
    income array Yes Array of income items (see below)

    Income Items

    Each income item in the array contains:

    Field Type Required Options/Description
    type string Yes Options: "salary", "allowance"
    subtype string Yes Options: "basic-salary", "housing-allowance", "transport-allowance", "leave-pay", "overtime-allowance", "directors-fee", "other-cash-benefits", "other-allowance"
    amount number Yes Amount for this income item
    metadata object No Additional data specific to the income type

    Benefits

    Field Type Required Description
    benefits array No Array of benefit items

    Benefit Items

    Field Type Required Options/Description
    type string Yes Must be "benefit"
    subtype string Yes Options: "car-benefit", "housing-benefit", "other-benefit"
    amount number Yes Benefit value amount
    metadata object Yes Required metadata varies by subtype

    Car Benefit Metadata

    Field Required Options
    type Yes "owned", "hired"
    body_type Yes "saloon", "station-wagon", "pickup", "van", etc.
    cc_rating Yes Engine CC (e.g., "1500", "3000")
    ownership_cost No Cost if owned
    hiring_cost No Monthly cost if hired

    Housing Benefit Metadata

    Field Required Options
    type Yes "employer_rented", "employer_owned", "employee_owned"

    Other Benefit Metadata

    Field Required Description
    meal_benefit No Meal benefit amount
    non_cash_benefit No Non-cash benefit amount
    other_taxable_benefit No Other taxable benefits

    Allowable Deductions

    Pensions

    Field Type Required Description
    pensions array No Registered pension schemes

    Each pension item:

    Field Required Description
    name Yes Pension scheme name
    scheme_id Yes Scheme registration ID
    amount Yes Monthly contribution amount
    type Yes Must be "pension"

    PRMFs (Provident Funds)

    Field Type Required Description
    prmfs array No Provident fund contributions

    Each PRMF item:

    Field Required Description
    name Yes Fund name
    scheme_id Yes Scheme registration ID
    amount Yes Monthly contribution amount
    type Yes Must be "PRMF"

    Mortgages

    Field Type Required Description
    mortgages array No Mortgage interest relief

    Each mortgage item:

    Field Required Description
    name Yes Lender name
    account_number Yes Mortgage account number
    amount Yes Monthly interest amount
    start_date Yes Mortgage start date (ISO 8601)
    end_date Yes Mortgage end date (ISO 8601)
    type Yes Must be "mortgage-interest"

    Insurances

    Field Type Required Description
    insurances array No Health/life insurance for relief

    Each insurance item should include premium amounts eligible for tax relief.

    Metadata

    Field Type Required Description
    metadata.bank_name string No Employee's bank name
    metadata.bank_code string No Bank code
    metadata.bank_account_number string No Account number
    metadata.disability_tax_exempt boolean No Whether employee qualifies for disability exemption
    metadata.disability_tax_exemption_no string No Exemption certificate number
    metadata.nita_registered boolean No NITA registration status

    Authentication

    Field Type Required Description
    employer_type string Yes organization or individual
    organization_id string Yes ID of currently authenticated organization
    employer_tax_payer_id string Yes ID of currently authenticated individual

    Calculated Fields

    These fields are computed by the system and should not be provided in the input:

    • gross_pay
    • taxable_pay
    • tax_due
    • net_pay
    • relief
    • deduction
    • allowance
    • benefit
    • items (detailed breakdown)
